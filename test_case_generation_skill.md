
# Task / Code → Test Case Generation Skill（AI-Optimized）

## 概要
タスクまたはコードから、網羅的なテストケースを生成するための skill。
AI活用を前提とし、「抜け漏れのないテスト設計」と「再現性のある品質担保」を目的とする。

---

## 目的
- 実装の正しさを検証する
- テスト観点の抜け漏れを防ぐ
- 正常系・異常系・境界値を網羅する
- 状態・権限・外部依存を考慮する
- 品質の再現性を担保する

---

## 入力
- タスク（Design → Task Breakdown の出力）
- コード（関数 / API / UIロジック）
- API仕様
- データモデル

---

## 出力（必須）
- テストケース一覧
  - 正常系
  - 異常系
  - 境界値
  - 状態遷移
  - 権限
  - 外部依存
  - 競合 / 重複
- 各テストケースの詳細
  - 前提条件
  - 入力
  - 期待結果
  - 実行手順

---

## 基本原則
- 正常系だけで終わらない
- すべての分岐をテスト対象にする
- 境界値を必ず含める
- 状態・権限を考慮する
- 外部依存の失敗も考慮する
- 仕様ベースで考える（実装依存にしない）
